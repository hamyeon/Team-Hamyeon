# Bunjang Crawling

번개장터 검색 API를 활용하여 상품 데이터를 수집하는 크롤링 테스트 코드입니다.

## 1️⃣ 검색 결과 수집

`get_bunjang_data` 함수는 사용자가 입력한 키워드를 기준으로 번개장터 검색 API를 호출한 뒤, 검색 결과 목록에서 상품 정보를 수집합니다.

```bash
python script.py
```

기본 실행 시 아래 설정값으로 동작합니다.

```python
SEARCH_KEYWORD = "나이키"
PAGE_COUNT = 2
```

수집되는 주요 정보는 다음과 같습니다.

1. 상품 ID (pid)
2. 상품명
3. 가격
4. 이미지 URL
5. 태그
6. 상태
7. 업데이트 시간
8. 지역

검색 결과는 CSV 파일로 저장되며, 파일명은 다음과 같습니다.

```
bunjang_<키워드>_results.csv
```

---

## 2️⃣ 데이터 수집 로직 특징

해당 스크립트는 단순 요청이 아닌 실제 서비스 환경을 고려한 크롤링 방식으로 구성되어 있습니다.

1. **광고 상품 필터링**

   * `ad=True`인 상품은 자동 제외

2. **이미지 URL 정제**

   * `{res}` 값을 `600`으로 치환하여 실제 이미지 URL 생성

3. **봇 차단 대응**

   * User-Agent 및 Referer 헤더 설정
   * 요청 간 랜덤 지연 (`2~4초`)

4. **페이지 단위 수집**

   * 페이지당 최대 100개 상품 수집
   * `PAGE_COUNT`로 전체 수집량 제어

---

## 3️⃣ 출력 데이터 활용

수집된 CSV 파일은 다음과 같은 용도로 활용할 수 있습니다.

1. 중고 상품 가격 분석
2. 유사 상품 매칭 데이터 구축
3. 이미지 기반 상태 분석 파이프라인 입력
4. 추천 가격 모델 학습 데이터

---

## 4️⃣ 주의사항

1. 본 코드는 비공식 API를 사용하므로, 서비스 구조 변경 시 동작이 중단될 수 있습니다.
2. 요청 빈도를 과도하게 높일 경우 IP 차단이 발생할 수 있습니다.
3. 실제 서비스 적용 시 번개장터 이용 약관(ToS)을 반드시 확인해야 합니다.
