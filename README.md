# 🛍️ AI 가격 검증 모델 기반 빈티지 거래 플랫폼

기존 빈티지 거래 시장에서는 같은 모델이라도 상태, 연식, 사이즈, 구성품 여부에 따라 가격 차이가 크게 발생합니다.

하지만 판매자가 가격을 임의로 설정하는 경우가 많아, 구매자는 해당 가격이 적정한지 판단하기 어렵습니다.

본 프로젝트는 이러한 문제를 해결하기 위해 다음과 같은 흐름으로 설계되었습니다.

- 상품 이미지와 메타데이터 기반 유사 거래 상품 탐색
- 거래 사례 기반 적정 가격 범위 추정
- 판매자 희망가와 AI 추정가 비교를 통한 가격 검증
- 검증 결과를 활용한 신뢰 기반 매물 관리

즉, 단순한 중고거래 서비스가 아니라 **AI가 가격 판단을 보조하는 빈티지 거래 플랫폼**을 목표로 합니다.

<br />

## 📌 Problem

### 1️⃣ 비정형 상품 정보

빈티지 신발은 단순히 모델명만으로 가격을 판단하기 어렵습니다.

같은 모델이라도 다음 요소에 따라 실제 거래 가치는 달라질 수 있습니다.

- 오염도
- 사용감
- 밑창 마모
- 구성품 유무
- 사이즈 희소성
- 보관 상태

기존 방식에서는 판매자가 직접 상태를 설명해야 하며, “사용감 있음”, “B급 상태”와 같은 표현만으로는 실제 감가 수준을 객관적으로 판단하기 어렵습니다.

### 2️⃣ 가격 정보의 비대칭성

판매자가 임의로 가격을 설정하면 구매자는 해당 가격이 적정한지 판단하기 어렵습니다.

가격이 너무 높으면 거래가 지연되고, 너무 낮으면 판매자가 상품 가치보다 낮은 가격에 판매할 수 있습니다.

따라서 상품 상태와 시세 데이터를 함께 반영한 가격 판단 기준이 필요합니다.

### 3️⃣ 탐색 및 입찰 피로도

구매자는 여러 매물의 상태와 가격을 직접 비교해야 합니다.

특히 경매 방식으로 확장될 경우, 구매자는 현재 입찰가가 적정한지, 더 입찰해도 되는지, 즉시 구매가가 합리적인지 계속 판단해야 합니다.

결국 빈티지 신발 거래의 핵심 문제는 단순히 상품을 등록하는 것이 아니라, 비정형 상품 정보를 구조화하고 시세와 비교하며 판매자와 구매자의 거래 판단을 보조하는 것입니다.

<br />

Vintic은 상품 이미지를 구조화하고, 시세 데이터와 비교하여 판매자와 구매자의 거래 판단을 보조합니다.

전체 흐름은 다음과 같습니다.

1. 사용자가 상품 이미지를 업로드합니다.
2. 서버는 업로드된 이미지를 S3에 저장합니다.
3. 저장된 이미지 URL을 기반으로 OpenAI Vision API가 상품 정보를 분석합니다.
4. AI는 브랜드, 모델명, 색상, 사이즈, 상태 설명 등을 추출합니다.
5. 사용자는 AI 분석 결과를 확인하고 수정할 수 있습니다.
6. 백엔드는 KREAM/eBay 유사 거래 데이터를 바탕으로 기준 시세를 계산합니다.
7. 상품 상태와 구성품 여부에 따라 보정률을 적용합니다.
8. 추천 판매가, 권장 가격 범위, 가격 산정 이유를 제공합니다.
9. 사용자가 최종 판매가를 정하면 상품 정보가 DB에 저장됩니다.

Vintic에서 AI의 역할은 가격을 직접 결정하는 것이 아닙니다.

AI는 비정형 이미지를 가격 계산에 사용할 수 있는 정형 상품 데이터로 변환하고, 최종 가격 판단은 시세 데이터와 백엔드 계산 로직을 통해 수행됩니다.

<br />

## 📌 Key Features

### 1️⃣ AI 기반 상품 정보 구조화

상품 이미지를 바탕으로 브랜드, 모델명, 색상, 사이즈, 상태 정보를 분석하고 가격 계산에 사용할 수 있는 정형 데이터로 변환합니다.

### 2️⃣ 시세 기반 가격 판단

KREAM/eBay 유사 거래 데이터를 활용해 기준 시세를 계산하고, 상품 상태와 구성품 여부를 반영해 추천 판매가와 권장 가격 범위를 제공합니다.

### 3️⃣ 판매자 스마트 등록

판매자는 AI가 분석한 상품 정보를 확인하고 수정한 뒤, 시세 기반 추천 가격을 참고해 더 빠르게 상품을 등록할 수 있습니다.

### 4️⃣ 구매자 조건 기반 탐색 및 입찰 보조

구매자는 원하는 브랜드, 사이즈, 예산, 상태 조건을 바탕으로 매물을 탐색할 수 있으며, 향후 AI Agent를 통해 입찰 판단 보조로 확장할 수 있습니다.

### 5️⃣ AI 판단 검증 및 안전장치

AI 분석 결과를 그대로 최종값으로 확정하지 않고, 사용자 확인과 백엔드 검증 구조를 통해 오판 가능성을 줄입니다.

### 6️⃣ 경매 시스템 확장 가능성

향후 실시간 경매, 자동 입찰 Agent, Redis/Kafka 기반 이벤트 처리, Harness Layer 검증 구조로 확장할 수 있습니다.

## 📌 Expected Impact

본 프로젝트는 구매자와 판매자 모두에게 더 신뢰할 수 있는 거래 환경을 제공하는 것을 목표로 합니다.

### 1. 구매자

- **상태 대비 가격이 합리적인지 더 빠르게 판단할 수 있습니다.**  
  상품 상태와 유사 거래 시세를 함께 반영한 추천 가격을 통해 매물의 가격 적정성을 더 쉽게 확인할 수 있습니다.

- **시세 탐색과 입찰 판단에 드는 피로를 줄일 수 있습니다.**  
  여러 플랫폼의 거래 사례를 반복적으로 비교하지 않아도 되고, 향후 조건 기반 입찰 보조 기능을 통해 원하는 조건의 매물을 더 효율적으로 탐색할 수 있습니다.

### 2. 판매자

- **상품 등록과 가격 책정 부담을 줄일 수 있습니다.**  
  AI가 상품 이미지를 분석하고, 시스템이 시세 기반 추천 가격을 제공하기 때문에 판매자는 더 빠르게 상품을 등록할 수 있습니다.

- **합리적인 가격 기준으로 거래를 시작할 수 있습니다.**  
  데이터 기반 가격 판단을 참고해 최종 판매가를 설정할 수 있어 과도하게 높거나 낮은 가격 책정 가능성을 줄일 수 있습니다.

### 3. 플랫폼

- **AI 기반 거래 판단 보조 구조를 구축할 수 있습니다.**  
  상품 분석, 시세 조회, 가격 판단, 최종 등록 흐름을 기반으로 거래 데이터를 누적하고, 향후 경매와 Agent 기능으로 확장할 수 있습니다.

- **빈티지 거래 시장의 정보 비대칭을 줄이는 기반이 됩니다.**  
  비정형 상품 정보를 구조화하고, 시세 기반 가격 판단을 제공함으로써 더 신뢰도 높은 거래 환경을 만들 수 있습니다.

## 📌 Data Collection Experiments

현재 저장소에는 가격 검증 모델의 입력 데이터를 확보하기 위한 **데이터 수집 테스트 코드**가 포함되어 있습니다.

이 코드는 완성된 서비스 기능이라기보다, **어떤 플랫폼에서 어떤 형태의 거래 데이터를 확보할 수 있는지 검증하기 위한 실험 코드**입니다.

### Kream

`kream-crawling/`은 크림 데이터를 수집하는 테스트 코드입니다.

검색 결과를 수집한 뒤, 그 결과를 바탕으로 상세 거래 데이터를 추가 수집하는 2단계 구조로 되어 있습니다.

- 검색 결과 수집
- 거래 이력 수집
- CSV 기반 후속 분석 가능

### eBay

`ebay-crawling/shoepricevalidator/`는 eBay 데이터를 활용한 백엔드 실험 코드입니다.

단순 수집을 넘어서 해외 시세 검색, 환율 반영, 가격 검증 API 구조까지 연결하는 프로토타입 성격을 가집니다.

- 해외 거래 시세 활용 가능성 검증
- 환율 및 검색어 처리 실험
- 가격 검증 서버 구조 실험

### Bunjang

루트의 `bunjang_crawler.py`는 번개장터 검색 결과를 수집하는 파이썬 스크립트입니다.

국내 개인 간 거래 플랫폼의 검색 결과 데이터를 빠르게 확보할 수 있는지 검증하기 위한 코드입니다.

- 검색어 기반 상품 리스트 수집
- 가격 및 상품 메타데이터 추출
- 국내 시세 표본 확보

## 📌 MVP

현재 MVP의 목표는 **AI Agent와 경매 시스템이 동작하기 위한 가격 판단 엔진의 기반**을 구현하는 것입니다.

전체 자동 입찰 Agent와 실시간 경매 시스템을 완성한 단계는 아니며, 상품 이미지 분석, 시세 조회, 가격 계산, 최종 상품 등록까지 이어지는 핵심 데이터 흐름을 구현했습니다.

### MVP Goal

빈티지 신발 거래에서 상품 이미지와 기본 정보를 바탕으로 추천 판매가와 권장 가격 범위를 제공하고, 최종 상품 등록까지 연결되는 데이터 흐름을 구현하는 것을 목표로 합니다.

### MVP Scope

1. 사용자가 상품 이미지를 여러 장 업로드합니다.
2. 서버가 업로드된 이미지를 S3에 저장합니다.
3. OpenAI Vision API가 브랜드, 모델명, 색상, 상태 정보를 분석합니다.
4. 사용자가 AI 분석 결과를 확인하고 수정합니다.
5. 시스템이 KREAM/eBay 유사 거래 데이터를 조회합니다.
6. 거래 데이터를 바탕으로 기준 시세와 추천 가격 범위를 산정합니다.
7. 사용자가 최종 판매가를 설정하고 상품을 등록합니다.
8. 등록된 상품을 상품 리스트에서 조회할 수 있습니다.

### MVP Output

- S3 이미지 URL
- AI 상품 분석 결과
- 브랜드 / 모델명 / 색상 / 사이즈 / 상태 등급
- KREAM 평균가
- eBay 평균가
- 기준 시세
- 추천 판매가
- 권장 가격 범위
- 가격 산정 이유
- 최종 등록 상품 정보

### What MVP Proves

이 MVP가 검증하려는 것은 단순히 서비스 화면을 만드는 것이 아니라,

**이미지 업로드부터 AI 분석, 시세 기반 가격 계산, 최종 상품 등록까지의 흐름이 하나의 파이프라인으로 연결될 수 있는가**입니다.

<br />

## 📌 Future Work

현재 MVP는 상품 분석과 가격 판단 기반 구현에 집중하고 있으며, 이후에는 다음과 같은 방향으로 확장할 수 있습니다.

- **AI Agent 고도화**: 판매자에게는 상품 특성에 따라 경매 시작가와 즉시 구매가를 제안하고, 구매자에게는 예산과 상태 조건에 맞는 입찰 판단을 보조하는 방식으로 확장할 수 있습니다.

- **경매 시스템 확장**: 실시간 입찰이 도입되면 Redis 기반 분산 락, 사용자 예산 잠금 로직, Kafka 기반 비동기 입찰 이벤트 처리, WebSocket 기반 실시간 입찰 상태 갱신을 적용할 수 있습니다.

- **Harness Layer 강화**: AI 분석 결과와 가격 산정 결과를 시세 데이터와 백엔드 규칙으로 검증하여 이상치를 감지하고, 사용자 확인이 필요한 경우를 분리할 수 있습니다.

- **구매자 조건 기반 입찰 보조**: 구매자가 원하는 브랜드, 사이즈, 가격대, 허용 가능한 상태 등급을 설정하면 AI Agent가 조건에 맞는 매물을 우선 입찰 후보로 판단하는 방식으로 확장할 수 있습니다.
